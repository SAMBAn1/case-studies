PILLAR 1 · PRIORITIZATION

"Who should matter more?"

Data-driven scoring per customer group. Default ranking mode is Group-Level Rank — sorted by ascending in-group rank, then by past-due amount as a tie-breaker.

Advanced option: Priority Band + Group Rank, for when groups use different scoring models and raw scores aren't comparable.

PILLAR 2 · ALLOCATION

"Who owns this customer?"

Fixed assignment rules by default (region, customer class, company code). Unmatched customers fall back to Shared Pool rather than a forced, arbitrary owner.

Bandwidth shapes Today's Focus fill — it never silently reassigns ownership.

PILLAR 3 · ACTION

"What should be done?"

Action Sequence Mode default: Highest Eligible Action Wins — the strongest matching action shows, even if earlier ones weren't completed.

Enterprise SOP-heavy customers can opt into Sequential Completion Required instead.

PILLAR 4 · WORKLIST

"What shows up today, and why?"

Today's Focus runs Full Portfolio with Focus Target — the whole eligible, assigned portfolio stays visible, with the top portion flagged as focus.

Cooldown suppresses without hiding: suppressed customers stay visible and filterable in Eligible Backlog.

This configuration view is read-only in the demo — in the real product, every value here is admin-editable with simulation before activation.
